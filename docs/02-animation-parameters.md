# 动画参数详解

## 概述

本文档详细描述从视频分析中提取的动画参数,包括时序、缓动函数和状态转换。

## 核心发现总结

| 特性 | 结论 |
|------|------|
| **动画类型** | 纯透明度动画 (opacity transition) |
| **位移** | 无 (卡片固定在 bottom: 0, left: 50%) |
| **分层结构** | 背景层 + 内容行 (独立控制) |
| **时序对称性** | 不对称 (淡出 5.0s / 淡入 1.4s) |
| **模糊效果** | 无 (锐度变化源于背景纹理) |
| **交互响应** | 淡出慢 (给用户消化时间) / 淡入快 (响应迅速) |

## 展开动画 (淡出)

### 时间轴总览

```
0s        2s        4s        6s
|---------|---------|---------|
▓▓▓▓▓▓▓▓▓▓░░░░░░▒▒▒░░░░░░░░░
   初始      第一行   第二行   背景
  静态       淡出     淡出    +剩余
```

### 分阶段参数

#### 阶段 1: 初始静态 (0-2.0s)

- **时长**: 2.0s
- **状态**: 完全可见
- **Content Ratio**: 0.905 (稳定)
- **目的**: 给用户充足时间阅读内容

#### 阶段 2: 第一行淡出 (2.0-2.2s)

| 参数 | 值 |
|------|---|
| **开始时间** | 2.0s |
| **时长** | 0.2s (200ms) |
| **目标元素** | `.card-content-line:first-child` |
| **起始 opacity** | 1.0 |
| **结束 opacity** | 0.0 |
| **缓动函数** | `power2.out` (慢速结束) |
| **Content Ratio** | 0.905 → 0.870 |

**缓动曲线**: ease-out (快速启动 → 平滑停止)

```
opacity
1.0 ┐
    │╲
    │ ╲___
    │     ╲___
0.0 └─────────╲
    2.0s     2.2s
```

#### 阶段 3: 静态保持 (2.2-3.85s)

- **时长**: 1.65s
- **状态**: 第一行已消失,其余可见
- **Content Ratio**: 0.869 (稳定)
- **目的**: 让用户适应第一行消失

#### 阶段 4: 第二行淡出 (3.85-4.0s)

| 参数 | 值 |
|------|---|
| **开始时间** | 3.85s |
| **时长** | 0.15s (150ms) |
| **目标元素** | `.card-content-line:nth-child(2)` |
| **起始 opacity** | 1.0 |
| **结束 opacity** | 0.0 |
| **缓动函数** | `power2.out` |
| **Content Ratio** | 0.869 → 0.620 (两步: 0.755 @ 3.85s → 0.620 @ 3.93s) |

#### 阶段 5: 静态保持 (4.0-4.5s)

- **时长**: 0.5s
- **状态**: 前两行已消失,剩余内容可见
- **Content Ratio**: 0.593 (稳定)

#### 阶段 6: 背景+剩余内容快速淡出 (4.5-5.0s)

| 参数 | 值 |
|------|---|
| **开始时间** | 4.5s |
| **时长** | 0.5s (500ms) |
| **目标元素** | 背景 (`backgroundRef`) + 剩余内容行 |
| **起始 opacity** | 1.0 |
| **结束 opacity** | 0.0 |
| **缓动函数** | `power2.in` (快速 ease-in) |
| **Content Ratio** | 0.593 → 0.025 (四步快速下降) |

**缓动曲线**: ease-in (慢速启动 → 快速结束)

```
opacity
1.0 ┐
    │    ╲
    │     ╲___
    │         ╲___
0.0 └─────────────╲
    4.5s         5.0s
```

**关键差异**: 比前两行快 (500ms vs 200ms/150ms),且使用 ease-in 而非 ease-out

#### 阶段 7: 完全透明 (5.0s+)

- **状态**: 完全不可见
- **Content Ratio**: 0.025-0.033 (残留背景噪声)

### GSAP 实现代码

```typescript
const timeline = gsap.timeline();

// 阶段 2: 第一行淡出 @ 2.0s
timeline.to(lineElements[0], {
  opacity: 0,
  duration: 0.2,
  ease: 'power2.out'
}, 2.0);

// 阶段 4: 第二行淡出 @ 3.85s
timeline.to(lineElements[1], {
  opacity: 0,
  duration: 0.15,
  ease: 'power2.out'
}, 3.85);

// 阶段 6: 背景+剩余内容淡出 @ 4.5s
timeline.to(backgroundRef.current, {
  opacity: 0,
  duration: 0.5,
  ease: 'power2.in'
}, 4.5);

timeline.to(Array.from(lineElements).slice(2), {
  opacity: 0,
  duration: 0.5,
  ease: 'power2.in'
}, 4.5); // 与背景同时开始
```

## 收起动画 (淡入)

### 时间轴总览

```
0s       1s       2s
|---------|---------|
▒▒▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
背景  静态  内容逐行淡入
快速  半透  (每行100ms)
```

### 分阶段参数

#### 阶段 1: 背景快速淡入 (0-0.1s)

| 参数 | 值 |
|------|---|
| **开始时间** | 0s (触发时立即开始) |
| **时长** | 0.1s (100ms) |
| **目标元素** | 背景 (`backgroundRef`) |
| **起始 opacity** | 0.0 |
| **结束 opacity** | 1.0 |
| **缓动函数** | `power2.out` |
| **Content Ratio** | 0.030 → 0.304 (单步快速上升) |

**缓动曲线**: ease-out (快速启动)

```
opacity
1.0 ┌───────
    │╱
   ╱│
  ╱ │
0.0 ╯
    0s  0.1s
```

**关键**: 极快的响应 (100ms = 6帧),给用户立即反馈

#### 阶段 2: 静态半透明 (0.1-1.0s)

- **时长**: 0.9s
- **状态**: 背景已显示,内容仍透明
- **Content Ratio**: 0.314 (稳定)
- **目的**: 建立空间锚点,为内容淡入做准备

#### 阶段 3: 内容逐行淡入 (1.0-1.4s)

| 参数 | 行 1 | 行 2 | 行 3 |
|------|------|------|------|
| **开始时间** | 1.0s | 1.1s | 1.2s |
| **时长** | 0.1s | 0.1s | 0.1s |
| **间隔** | - | 0.1s | 0.1s |
| **起始 opacity** | 0.0 | 0.0 | 0.0 |
| **结束 opacity** | 1.0 | 1.0 | 1.0 |
| **缓动函数** | `power2.out` | `power2.out` | `power2.out` |

**Content Ratio 变化**:
- 1.0s: 0.314 → 0.593 (行 1 淡入)
- 1.1s: 0.593 → 0.672 (行 2 淡入)
- 1.3s: 0.672 → 0.849 (行 3 淡入)
- 1.4s: 0.849 → 0.970 (完全可见)

**视觉效果**: 从上到下依次浮现 (级联动画)

```
时间轴:
1.0s   1.1s   1.2s   1.3s
|------|------|------|
行1 ████
行2    ████
行3       ████
```

#### 阶段 4: 完全可见 (1.4s+)

- **状态**: 所有内容完全显示
- **Content Ratio**: 0.966-0.970 (稳定)

### GSAP 实现代码

```typescript
const timeline = gsap.timeline();

// 设置初始状态 (所有元素透明)
gsap.set(backgroundRef.current, { opacity: 0 });
gsap.set(lineElements, { opacity: 0 });

// 阶段 1: 背景淡入 @ 0s
timeline.to(backgroundRef.current, {
  opacity: 1,
  duration: 0.1,
  ease: 'power2.out'
}, 0);

// 阶段 3: 内容逐行淡入 @ 1.0s, 1.1s, 1.2s
lineElements.forEach((line, index) => {
  timeline.to(line, {
    opacity: 1,
    duration: 0.1,
    ease: 'power2.out'
  }, 1.0 + (index * 0.1));
});
```

## 缓动函数对比

### power2.out (展开第一、二行)

```
进度 vs 时间 (归一化)
1.0 ┐
    │╲___
    │    ╲___
0.5 │       ╲___
    │          ╲___
0.0 └──────────────╲
    0%           100%
```

**特点**: 快速启动 (前 20% 完成 50% 进度) → 平滑停止

**适用场景**: 内容淡出,避免突兀消失

### power2.in (展开背景+剩余)

```
进度 vs 时间 (归一化)
1.0 ┌──╱
    │ ╱
    │╱___
0.5 │    ╲___
    │       ╲___
0.0 └──────────────╲
    0%           100%
```

**特点**: 慢速启动 → 快速结束 (后 20% 完成 50% 进度)

**适用场景**: 最终淡出,快速彻底消失

### power2.out (收起所有元素)

**统一使用 ease-out**: 快速响应,所有元素同步淡入风格

## 时序设计哲学

### 不对称设计原理

| 维度 | 展开 (淡出) | 收起 (淡入) | 原因 |
|------|-------------|-------------|------|
| **总时长** | 5.0s | 1.4s | 淡出需消化时间 / 淡入需快速响应 |
| **分阶段** | 3 阶段 (逐行) | 2 阶段 (背景+内容) | 淡出强调层次 / 淡入强调整体 |
| **缓动** | out → in | 统一 out | 淡出渐变 → 加速 / 淡入保持快节奏 |
| **延迟** | 长 (1.65s, 0.5s) | 短 (0.9s) | 淡出给反应时间 / 淡入无需等待 |

### 用户体验目标

1. **淡出 (展开)**:
   - 第一行消失 → 用户注意到变化
   - 静态 1.65s → 用户确认第一行已消失
   - 第二行消失 → 确认趋势
   - 静态 0.5s → 准备最终消失
   - 背景消失 → 完全淡出

2. **淡入 (收起)**:
   - 背景 100ms → 立即给出反馈 ("已响应")
   - 静态 900ms → 建立空间 (避免内容与背景同时出现的混乱)
   - 内容 100ms/行 → 清晰的级联浮现

## 参数调整建议

### 加快淡出 (更紧凑)

```typescript
// 原: 2.0s, 3.85s, 4.5s
// 新: 1.0s, 2.0s, 2.5s (减半)
timeline.to(lineElements[0], { ... }, 1.0);   // 从 2.0 → 1.0
timeline.to(lineElements[1], { ... }, 2.0);   // 从 3.85 → 2.0
timeline.to(backgroundRef.current, { ... }, 2.5); // 从 4.5 → 2.5
```

### 加快淡入 (更即时)

```typescript
// 原: 0.1s, 0.9s 延迟, 0.1s/行
// 新: 0.05s, 0.2s 延迟, 0.05s/行 (减半)
timeline.to(backgroundRef.current, { duration: 0.05, ... }, 0);
lineElements.forEach((line, index) => {
  timeline.to(line, { duration: 0.05, ... }, 0.2 + (index * 0.05));
});
```

### 同步淡入 (无级联)

```typescript
// 背景和内容同时淡入
timeline.to(backgroundRef.current, { ... }, 0);
timeline.to(lineElements, { ... }, 0); // 所有行同时
```

### 使用其他缓动

```typescript
// 弹性淡入 (bounce 效果)
ease: 'back.out(1.7)'  // 轻微回弹

// 更平滑的淡出
ease: 'power4.out'     // 更柔和的曲线

// 线性 (匀速)
ease: 'none'           // 无缓动
```

## 与 CSS transition 对比

| 特性 | CSS Transition | GSAP Timeline |
|------|---------------|---------------|
| **时间控制** | 相对 (触发时开始) | 绝对 (2.0s, 3.85s, 4.5s) |
| **分层动画** | 需多个 class + delay | 单个 timeline 统一管理 |
| **暂停/恢复** | 不支持 | `timeline.pause()` / `resume()` |
| **逆向播放** | 不支持 | `timeline.reverse()` |
| **调试** | 需浏览器 DevTools | `timeline.time(2.0)` 跳转时间点 |
| **性能** | GPU 加速 (相同) | GPU 加速 (相同) |

**结论**: GSAP 更适合复杂时序,CSS 适合简单状态切换

## 数据溯源

所有参数来自 [01-video-analysis.md](./01-video-analysis.md) 的测量数据:

- **展开**: 70 个采样点 (0-5.8s, 每 83ms)
- **收起**: 30 个采样点 (16.7-19.2s, 每 83ms)
- **精度**: ±5 帧 = ±83ms
- **工具**: OpenCV Content Ratio + Laplacian Sharpness

## 总结

这是一个精心设计的**不对称透明度动画**:
- **展开**: 慢速分层淡出 (5.0s, 3 阶段, 2 种缓动)
- **收起**: 快速级联淡入 (1.4s, 2 阶段, 1 种缓动)
- **无位移**: 固定位置,纯 opacity 变化
- **用户体验**: 淡出给消化时间,淡入给即时反馈
