# SensorDeck 网站多视角审查报告

**审查日期：** 2026-07-21  
**审查范围：** 代码质量、业务转化、用户体验  
**审查方法：** 三个独立 Codex Agent 分别从工程师、企业采购、UX 设计师视角审查

---

## 📊 执行摘要

| 视角 | 关键发现 | 优先级分布 |
|------|----------|------------|
| **客户/采购** | 缺少客户案例、真实数据、安全材料、可靠联系方式 | Must Fix: 12 / Should Fix: 10 / Nice to Have: 3 |
| **工程师** | 全站动态 SSR、联系表单无后端、运行时翻译、图片未优化 | Critical: 0 / High: 6 / Medium: 9 / Low: 2 |
| **UX/设计** | 失效链接、术语过载、移动端首屏混乱、导航不清晰 | Blocks User: 3 / Confusing: 12 / Polish: 3 |

## 🔴 P0 级问题（阻断转化）

### BIZ-006 / UX-001：隐私政策路由不存在 + 关键链接失效
- **问题**：联系表单指向 `/privacy` 返回 404；首页"2 分钟概览"指向不存在的锚点
- **影响**：法务阻止提交信息；高意向访客流失
- **修复**：创建隐私政策页面；修复所有断链

### BIZ-005 / ENG-002 / UX-002：联系表单无服务端提交
- **问题**：仅构造 `mailto:` 链接，依赖本地邮件客户端
- **影响**：移动端/企业环境失败；无法跟踪销售线索
- **修复**：接入 Server Action 或 API Route；提供提交状态和失败重试

### BIZ-004：销售团队显示占位文案
- **问题**：联系页显示"销售联系方式和团队信息将由 SensorDeck 更新"
- **影响**：降低商业准备度可信度
- **修复**：移除占位内容或提供真实联系人信息

### UX-003：文字对比度不足
- **问题**：`text-ink/40`（对比度 2.54:1）、页脚 `text-white/45`（4.44:1）低于 WCAG AA 标准（4.5:1）
- **影响**：低视力用户无法阅读关键信息
- **修复**：提高透明度至至少 60%；辅助文字最小 12px

## 🟠 P1 级问题（严重影响体验）

### ENG-001：全站强制动态 SSR
- **问题**：根布局调用 `headers()` 读取路径，导致所有页面无法静态生成
- **影响**：TTFB 慢、CDN 无法缓存、服务器成本高
- **修复**：将 `<html lang>` 移入 `[lang]/layout.tsx`；由代理层处理根路径跳转

### ENG-003：运行时递归翻译克隆整棵 React 树
- **问题**：每次渲染执行 `Object.entries().sort()` + 递归克隆，279 行翻译表在 4 个 demo 中重复调用
- **影响**：交互卡顿、渲染性能差
- **修复**：删除 `legacy-i18n.tsx`；直接使用类型化内容模块

### ENG-004：首页图片 2.6MB+ 未优化
- **问题**：桌面+移动首屏图同时渲染，`quality={100}` + `unoptimized`
- **影响**：LCP 长、移动流量消耗大
- **修复**：转为 AVIF/WebP；用 `<picture>` 做 art direction；移除 `unoptimized`

### ENG-005：联系页 377 行全部客户端渲染
- **问题**：整个页面包裹在 `"use client"` 组件内
- **影响**：JS 体积大、水合成本高、违背 RSC 架构
- **修复**：仅表单交互部分为客户端岛；页面主体保持 Server Component

### ENG-006：Demo 无障碍性缺失
- **问题**：`<label>` 无 `htmlFor`、控件无 `id`、分段按钮无 ARIA 语义
- **影响**：屏幕阅读器、键盘用户无法使用
- **修复**：建立 label 关联；分段控件用 `fieldset` 或 `aria-pressed`

### BIZ-001：无客户案例或社会证明
- **问题**：网站看不到任何真实客户、试点、合作伙伴
- **影响**：无法判断产品是否经过验证，阻碍供应商入围
- **修复**：发布 2-3 个案例（匿名可），包含客户类型、部署规模、实施周期、结果

### BIZ-002：所有 ROI 数据为示意数据
- **问题**：无法向管理层申请预算
- **修复**：提供带统计口径的真实指标（样本量、时间范围）

### BIZ-003：公司背景信息缺失
- **问题**：无成立时间、法律主体、团队背景、融资、知识产权
- **影响**：无法完成供应商背调
- **修复**：补充可核实的公司资质（不宜公开时至少说明法律实体）

### BIZ-007：白皮书全部标记"即将推出"
- **问题**：6 份白皮书均不可下载；首页卡片带箭头但不可点击
- **影响**：价值承诺与实际不符，中断技术尽调
- **修复**：至少开放一份核心白皮书和一个示例 Evidence Pack

### BIZ-009：缺少部署和集成技术规格
- **问题**：无 CPU/内存开销、支持的 ROS 版本、传感器接口、API 范围
- **影响**：无法评估集成成本和兼容风险
- **修复**：发布部署架构、兼容矩阵、资源基准、接口清单

### BIZ-012：无安全材料
- **问题**：无加密、RBAC、审计、租户隔离、漏洞管理信息
- **影响**：信息安全团队无法评估，可能直接否决 PoC
- **修复**：提供 Security Overview、数据流图、访问控制、合规状态

### UX-004：多处"申请演示"指向不同流程
- **问题**：导航栏→联系页，首屏→角色 demo 页
- **影响**：用户困惑，不知哪个是真正预约入口
- **修复**：统一指向联系表单；产品演示改名"体验交互演示"

### UX-006：专业缩写密度过高
- **问题**：REF、RGA、EGP、IR/LL、EP、Assist Vault 同时出现
- **影响**：非技术决策者难以理解价值
- **修复**：首次使用"中文含义（缩写）"；首页只保留 2-3 个核心概念

### UX-008：移动端首屏信息竞争严重
- **问题**：标题、说明、价值主张、3 个按钮、机器人背景小字同时争夺注意力
- **影响**：难以聚焦关键信息
- **修复**：移动端只保留主标题 + 一句说明 + 一个主 CTA

## 🟡 P2 级问题（影响质量）

### 工程类

**ENG-007**：站内导航用 `<a>` 而非 `next/link`  
→ 页面切换慢、无预取

**ENG-008**：`AnimatedSection` SSR 初始状态 `opacity-0`  
→ 首屏空白、JS 失败时不可见

**ENG-009**：Header scroll 监听触发高频状态更新 + 动画 `min-height`  
→ 滚动帧率下降、布局重绘

**ENG-010**：Demo 固定布局无移动降级（`grid-cols-5` 强制）  
→ 小屏文字压缩、横向溢出

**ENG-011**：内容加载器大量类型断言绕过校验  
→ 翻译字段漂移风险

**ENG-012**：无测试覆盖  
→ 回归风险高

**ENG-013**：`[lang]/layout.tsx` 只返回 Fragment，Header/Footer 在 20 个页面重复  
→ 模块化差、改动成本高

**ENG-014**：语言切换器 `role="menu"` 但无键盘导航  
→ 键盘用户体验差

**ENG-015**：无 `error.tsx`、`loading.tsx`、`not-found.tsx`  
→ 故障恢复弱、无可观测性

### 业务类

**BIZ-008**：演示视频链接指向不存在的 `/demo/overview`  

**BIZ-010**：产品线购买主体不清晰（OEM vs 传感器厂商）

**BIZ-011**：EGP 术语不一致（Engagement Pack / Evidence Governance Protocol）

**BIZ-013**：术语密度高 + 中英混排

**BIZ-014**：无竞品对比（vs 日志平台 / rosbag / 可观测性平台）

**BIZ-015**：强技术声明无适用条件（"7×24 观察""无需更换传感器"）

**BIZ-016**：无标准 PoC 范围和商业模式

**BIZ-017**：CTA 泛化，无角色化分流

**BIZ-018**：页脚无销售邮箱和响应时间

**BIZ-021**：中文术语不够自然（"运行时表面""宪法边界""候选证据包"）

**BIZ-022**：无 SLA、支持模式、培训、数据迁移说明

### UX 类

**UX-005**：白皮书卡片有箭头但不可点击

**UX-007**：首屏图片语言未随网站语言切换

**UX-009**：首屏 3 个高权重 CTA（演示 / 架构 / 白皮书）

**UX-010**：首页过长（11 个段落）且卡片结构重复

**UX-011**："产品 / 平台 / 技术"边界不清

**UX-012**：1024-1200px 笔记本也隐藏桌面导航

**UX-013**：导航无当前页面高亮

**UX-015**：联系表单字段过多（8 个字段）
